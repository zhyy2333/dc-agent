## 遇到的问题

1. **Playwright Sync API inside asyncio loop**（4 次尝试解决）
   - 原因：Anthropic SDK / VSCode / 宿主环境可能创建 asyncio 事件循环，Playwright sync API 检测到后拒绝运行
   - 尝试 1：ThreadPoolExecutor + @_in_browser_thread 装饰器 — Windows 上未生效
   - 尝试 2：独立后台线程 + asyncio.new_event_loop() + async_playwright + run_coroutine_threadsafe() — 成功隔离，冒烟测试通过

2. **BOSS 直聘反自动化检测**
   - 现象：登录页加载后跳回 about:blank 空白页
   - 解决 1：添加 --disable-blink-features=AutomationControlled 等反检测参数
   - 解决 2（最终方案）：改用 CDP 连接用户真实 Chrome，Playwright 只做操控不做启动，完全绕过检测

3. **登录流程反复跳转**
   - 现象：boss_check_login 内部 page.goto() 把用户正在扫码的登录页跳走
   - 解决：删除 boss_check_login 中的导航逻辑，改为 CLI 层用 boss_navigate 显式控制每一步

4. **async API 参数差异**：chromium.launch() 不支持 user_data_dir，改用 launch_persistent_context()
5. **IPv6 localhost**：Windows 上 localhost 解析为 ::1 导致 CDP 连接拒绝，改用 127.0.0.1
